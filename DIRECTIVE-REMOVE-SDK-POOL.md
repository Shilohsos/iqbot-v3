# Issue: SDK Connection Pool causes persistent "authentication is failed" errors

## Problem

The SDK connection pool (`src/sdkpool.ts`) keeps persistent WebSocket connections alive for each SSID. Multiple functions (`analyzePair`, `executeTrade`, balance fetch, /pairs) share the pool, but IQ Option's server silently invalidates persistent connections or rejects the second concurrent connection for the same SSID. This causes "authentication is failed" errors on subsequent SDK calls — both during analysis and balance fetch.

## Evidence

- "Analysis failed: authentication is failed" on every trade attempt
- Balance line missing from home menu (balance fetch through pool fails silently)
- Errors persist despite fixes to `executeTrade` and `analyzePair` to use the pool
- Symptoms started when the pool was introduced

## Root Cause

The SDK pool keeps a WebSocket open for up to 5 minutes per SSID. IQ Option's infrastructure:
1. May close idle WS connections silently
2. Does not reliably handle multiple concurrent WS connections for the same SSID
3. Returns "authentication is failed" when using a stale/invalidated connection

## Required Fix: REMOVE THE SDK POOL entirely

Every SDK operation should create its own fresh WebSocket connection, use it, then shut it down. This is slower but reliable — it worked before the pool was introduced.

### Files to modify

#### 1. `src/sdkpool.ts` — DELETE this file entirely

#### 2. `src/trade.ts`
- Remove `import { getSdk, evictSdk } from './sdkpool.js'`
- Revert `executeTrade()` to use `ClientSdk.create()` with `sdk.shutdown()` in finally block
- Keep `createSdk()` export — it's still used by analysis.ts

#### 3. `src/analysis.ts`
- Remove `import { ClientSdk } from './index.js'` and `import { getSdk, evictSdk } from './sdkpool.js'`
- Add back `import { createSdk } from './trade.js'`
- Revert to: `const sdk = await createSdk(ssid)` with `sdk.shutdown()` in finally block

#### 4. `src/bot.ts`
- Remove `import { getSdk, evictSdk, runSdkOp } from './sdkpool.js'`
- Add `createSdk` to the import from `'./trade.js'`
- In `loginAndCaptureSsid()`: remove `evictSdk(ssid)`, change `getSdk(ssid)` back to `createSdk(ssid)`, remove `runSdkOp` wrapper
- In the bg balance fetch (`sendStartMenu` at line 434): change `getSdk(ssid!)` to `createSdk(ssid!)`, remove `runSdkOp` wrapper, add `await sdk.shutdown()` after the balance is used
- In the trade execution (line 577): remove `runSdkOp` wrapper
- In `/balance` command (line 1143): change `getSdk(ssid)` to `createSdk(ssid)`, remove `runSdkOp` wrapper, add `await sdk.shutdown()` after balance fetch
- In `/pairs` command (line 1767): change `getSdk(ssid)` to `createSdk(ssid)`, remove `runSdkOp` wrapper, add `await sdk.shutdown()` after usage
- In `/refresh` command (line 1795): remove `evictSdk(refreshSsid)` — no pool to evict

#### 5. `dist/analysis.js` — must be regenerated by running `npx tsc` after the source changes

### Caller pattern (replace ALL getSdk usages)

Replace EVERY instance of:
```typescript
const sdk = await runSdkOp(() => withTimeout(getSdk(ssid), N_ms, 'label'));
// ... use sdk ...
```

With:
```typescript
const sdk = await withTimeout(createSdk(ssid), N_ms, 'label');
try {
    // ... use sdk ...
} finally {
    await sdk.shutdown();
}
```
